# Architecture Stage

## Purpose

Record technical decisions that materially affect implementation, operations, or future change.

## Inputs

- Requirements.
- Harness standards.
- Offline/on-prem constraints.

## Outputs

- SDD.
- ADRs for key choices and exceptions.
- Data, security, integration, and deployment notes.

## Gate

Proceed only when infrastructure exceptions, data strategy, security boundaries, and deployment impact are explicit.
